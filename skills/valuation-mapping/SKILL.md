# 估值映射 Skill

## 0. 核心原则

不要直接用 PE、PS、PB、PEG 机械判断贵不贵。

必须先回答：

```text
市场现在到底在交易什么核心变量？
```

然后用当前市值反推：

```text
这个市值隐含了多少收入、利润、销量、价格、ROE、订单、产能、客户数或产品周期？
```

本 Skill 的目标不是写一篇普通研报，而是把股价背后的市场预期翻译成可验证、可反证的产业变量。

典型映射例子：

| 公司/行业 | 交易锚 |
|---|---|
| 牧原股份 | 头均利润、猪价、出栏量、成本 |
| 中芯国际 | 晶圆 ASP、产能利用率、制程结构、资本开支 |
| 铖昌科技 | 卫星颗数、T/R组件价值量、订单节奏 |
| 新华保险 | PEV、NBV、投资收益率、ROE修复 |
| 风华高科/MLCC | 稼动率、ASP、库存周期、ROE |
| 药明康德/CDMO | 外需订单、产能利用率、利润率、海外政策折价 |
| 华虹公司 | 晶圆 ASP、产能利用率、成熟制程景气、折旧压力 |
| 江化微/半导体材料 | 国产替代率、客户认证、电子化学品放量、利润率 |
| 玻璃基板 | 客户认证、月产片数、TGV设备订单、单片价值量、良率 |
| 创新药 | 峰值销售、授权首付款/里程碑、适应症空间、临床成功率 |
| 保险股 | PEV、NBV、投资收益率、ROE、分红能力 |
| 周期制造 | 单位利润、价格弹性、库存周期、产能利用率 |
| AI软件 | 收入增速、客户数、ARR、净留存率、海外可比估值 |

## 1. 触发条件

当用户出现以下表达时调用本 Skill：

- “用估值映射分析 XXX”
- “XXX 当前价格在交易什么”
- “这个市值隐含什么预期”
- “现在多少市值反映多少利润/收入/销量”
- “这个价格对应多少猪价/多少晶圆 ASP/多少订单/多少颗卫星/多少产能”
- “有没有类似牧原头均利润、中芯晶圆 ASP 这种映射”
- “市场在交易什么预期差”
- “国外可比公司估值怎么看”
- “贵不贵”
- “为什么这个估值合理”
- “这个票要突破需要做到什么”
- “当前股价是不是已经交易乐观情景”

如果用户只是问公司做什么，可以先做业务拆解，但不要完整跑估值映射。  
如果用户问的是行业而不是个股，优先使用 `industry-research`。  
如果用户问成长股估值贵不贵，可联动 `tam-adj-peg`。  
如果用户问走势是否健康，可联动 `gf-dma-health-index`。  
如果用户问海外大涨大跌对 A 股影响，可联动 `overseas-a-share-mapping`。

## 2. 必须先做当前数据校验

如果涉及当前股价、市值、财务、估值、公告、新闻、行业数据、海外可比公司估值，必须先获取最新数据。

不得凭记忆给出当前市值、股价、PE、PS、PB、订单、盈利预测、行业价格、海外可比估值。

需要校验的数据包括：

- 当前股价和总市值
- 最新财报收入、利润、毛利率、现金流、负债
- 一致预期收入、利润、增速
- 当前 PE、PB、PS、EV/EBITDA、股息率等估值指标
- 行业价格、销量、库存、产能、稼动率
- 公司公告、调研纪要、互动易、业绩预告、订单公告
- 海外可比公司当前估值和成长状态
- 最近 1-3 个月是否有政策、客户、订单、价格、产能、业绩预告催化

如果数据无法获取，必须明确说明数据缺口，并用情景假设替代，不能假装确定。

# 一、结论先行

先用 3-5 句话说明：

1. 当前股价/市值主要在交易什么。
2. 当前预期属于低、中、高哪一档。
3. 最大预期差在哪里。
4. 最关键的验证指标是什么。
5. 结论是低估、合理、偏贵，还是只能当主题期权。

示例：

```text
当前市场不是在交易传统 PE，而是在交易 XXX 的产业变量。当前市值大致隐含 XXX 收入/利润/销量/价格，属于中性偏乐观预期。要进一步突破，需要看到 XXX。若 XXX 不兑现，当前估值存在回撤风险。
```

结论要避免空话。不要只说“长期看好”“受益于 AI”“估值合理”。  
必须把结论落到可计算、可验证的变量上。

# 二、公司业务拆解

必须拆清楚公司业务，不得只按传统行业标签判断。

输出：

| 业务 | 收入/利润占比 | 增速 | 毛利率 | 市场现在是否交易这块 |
|---|---:|---:|---:|---|

重点判断：

- 公司的真实利润来源是什么？
- 市场现在交易的是老业务还是新业务？
- 是否存在跨行业重估？
- 是否有业务占比很小但市值弹性很大的新叙事？
- 当前市值主要由哪块业务解释？
- 如果新业务还没有收入，市场是否已经给了期权价值？

常见错误：

- 把传统主业和新叙事混在一起。
- 只看公司名称，不看利润来源。
- 把“有布局”当成“有利润”。
- 把小业务的远期空间直接映射到全公司市值。

# 三、交易锚识别

必须明确当前股价在交易的核心锚。

可能的交易锚包括：

| 类型 | 例子 |
|---|---|
| 商品价格 | 猪价、锂价、硅片价格、MLCC ASP、电子布价格 |
| 单位盈利 | 头均利润、单片盈利、单瓦盈利、单吨盈利、单台设备利润 |
| 产能/销量 | 出栏量、晶圆产能、设备台数、卫星颗数、机器人销量 |
| 订单 | CDMO订单、军工订单、机器人订单、半导体设备订单 |
| 客户认证 | 英伟达、苹果、TSMC、Intel、三星、国产大客户供应链 |
| 渗透率 | 国产替代率、技术路线渗透率、AI渗透率 |
| ROE/PB | 金融、保险、周期制造、重资产制造 |
| 峰值销售 | 创新药、医疗器械、消费品大单品 |
| 资本开支 | 半导体设备、AI基础设施、存储扩产、光模块扩产 |
| 海外映射 | 美股龙头估值、海外订单、产业链涨价、海外产能紧缺 |
| 政策折价/溢价 | 出口管制、医保控费、创新药出海、地缘风险 |

输出格式：

```text
当前股价的核心交易锚是：XXX。
不是在交易：XXX。
原因是：XXX。
```

必须区分：

- 市场表面叙事
- 真正利润来源
- 股价实际交易变量
- 可验证产业指标

# 四、市值反推

这是本 Skill 的核心。

必须用当前市值反推市场隐含预期。

## 4.1 基础拆分

先拆：

```text
当前总市值 = 老业务合理价值 + 新业务隐含价值 + 主题溢价/折价
```

如果公司多业务并存，必须做 SOTP：

```text
合理市值 = 业务A价值 + 业务B价值 + 业务C价值 - 净负债/折价项
```

输出表：

| 项目 | 数值/假设 |
|---|---:|
| 当前总市值 | |
| 净现金/净负债 | |
| 老业务合理价值 | |
| 新业务隐含市值 | |
| 当前市场核心定价部分 | |
| 合理估值倍数 | |
| 反推隐含收入 | |
| 反推隐含利润 | |
| 反推隐含销量/价格/订单/ROE | |
| 当前真实水平 | |
| 差距 | |

## 4.2 常用反推公式

### 1. 收入型

```text
隐含收入 = 当前业务对应市值 / 合理 PS
```

适用于：

- SaaS
- 半导体材料
- 早期成长股
- 尚未稳定盈利的新业务
- 高增长但利润暂时被研发/产能压制的公司

注意：

- PS 不能拍脑袋。
- 必须参考海外可比、毛利率、增长率、利润率、市场空间。
- 高 PS 需要高增长、高毛利、高确定性支撑。

### 2. 利润型

```text
隐含净利润 = 当前业务对应市值 / 合理 PE
```

适用于：

- 已经盈利且利润可持续的公司
- 成熟制造
- 金融
- 周期股中后段
- 稳定现金流公司

注意：

- 不能用周期顶部利润给高 PE。
- 不能用一次性利润当正常利润。
- 要区分净利润和扣非净利润。

### 3. ROE-PB 型

```text
合理 PB ≈ ROE / 股权成本
隐含 ROE ≈ PB × 股权成本
```

适用于：

- 银行
- 保险
- 证券
- 重资产制造
- 周期资产
- 资产负债表驱动型公司

输出必须回答：

- 当前 PB 隐含什么 ROE？
- 当前 ROE 是否可持续？
- ROE 上修来自利润率、周转率、杠杆，还是资产重估？
- 海外可比公司 PB-ROE 对应关系如何？

### 4. 单位盈利型

```text
隐含单位盈利 = 隐含净利润 / 销量
```

适用于：

- 养殖：头均利润
- 硅片/晶圆：单片盈利
- 光伏：单瓦盈利
- 化工：单吨盈利
- 设备：单台盈利
- 机器人：单机盈利

必须回答：

- 当前单位盈利是多少？
- 市值隐含单位盈利是多少？
- 这个单位盈利在历史周期中处于什么分位？
- 能否由价格、成本、利用率共同解释？

### 5. 销量/订单型

```text
隐含销量 = 隐含收入 / 单价
隐含订单 = 隐含收入 / 确认周期
```

适用于：

- 军工
- 卫星
- 半导体设备
- 机器人
- 大客户订单
- 数据中心设备

必须回答：

- 市场隐含了多少订单？
- 订单是否已有公告或产业验证？
- 收入确认周期多长？
- 订单转利润率是多少？

### 6. 创新药峰值销售型

```text
合理市值 = 峰值销售 × 净利率 × PE × 成功率折现
```

或：

```text
合理市值 = 峰值销售 × P/S × 成功率折现
```

适用于：

- 创新药
- ADC
- GLP-1
- 双抗
- 小核酸
- license-out 类公司

必须回答：

- 适应症空间多大？
- 峰值销售假设是否合理？
- 竞争格局如何？
- 临床阶段对应成功率折现多少？
- 是否已有 BD、首付款、里程碑验证？
- 当前市值是否已经提前交易峰值销售？

### 7. 设备/军工/卫星型

```text
隐含收入 = 产品数量 × 单产品价值量
隐含利润 = 隐含收入 × 净利率
```

适用于：

- 卫星链
- 军工电子
- 相控阵雷达
- 半导体设备
- 机器人零部件
- AI服务器零部件

必须回答：

- 单产品价值量是多少？
- 总需求数量是多少？
- 公司份额是多少？
- 毛利率/净利率是多少？
- 当前市值隐含多少数量？
- 真实订单和产能是否支持？

# 五、三档情景

必须给出保守、中性、乐观三档。

| 情景 | 核心假设 | 收入 | 净利润 | 合理估值 | 对应市值 | 对当前价格判断 |
|---|---|---:|---:|---:|---:|---|
| 保守 | | | | | | |
| 中性 | | | | | | |
| 乐观 | | | | | | |

要求：

- 保守情景不能过度悲观，要能对应真实下限。
- 乐观情景不能纯拍脑袋，要有产业或公司验证路径。
- 中性情景应该最接近当前可验证事实。
- 必须说明当前市值更接近哪一档。
- 必须说明市场是否已经交易乐观情景。
- 必须说明股价要进一步突破需要哪项数据上修。

情景要和交易锚一致。

例如：

- 牧原：猪价、出栏量、成本、头均利润。
- 中芯：ASP、产能利用率、折旧、资本开支。
- 创新药：峰值销售、成功率、BD金额、上市节奏。
- 玻璃基板：客户认证、产能、良率、单片价值量。
- 半导体设备：订单、收入确认、国产替代率、净利率。

| 毛利率 | | | |
| 净利率 | | | |
| ROE | | | |
| PE/PS/PB/EV-EBITDA | | | |
| 市场空间 | | | |
| 客户结构 | | | |
| 技术壁垒 | | | |
| 估值溢价/折价原因 | | | |

必须回答：

- 海外可比是否真可比？
- 本公司应该折价还是溢价？
- 折价来自政策、客户、技术、规模、治理、流动性还是成长确定性？

- 海外估值是否本身也处于泡沫或低谷？

禁止只列公司名字，不做估值和商业模式比较。

# 七、预期差

必须判断市场预期和事实之间的差距。

输出：

| 维度 | 市场当前预期 | 实际情况 | 预期差方向 |
|---|---|---|---|
| 收入 | | | |
| 利润 | | | |
| 订单 | | | |
| 价格 | | | |
| 产能 | | | |
| 客户 | | | |
| 政策/监管 | | | |
| 海外映射 | | | |
| 财报验证 | | | |

预期差分三类：

1. 正预期差：事实好于市场认知。
2. 负预期差：市场已经过度乐观。
3. 时间差：逻辑对，但兑现太慢。

必须回答：

- 市场是不是已经知道这个逻辑？
- 当前股价是否已经反映？
- 哪个数据会让市场上修？
- 哪个数据会让市场下修？
- 预期差是 1 个月内能验证，还是 1 年后才能验证？

# 八、兑现路径和时间维度

必须说明逻辑如何兑现。

| 时间 | 关键节点 | 验证指标 |
|---|---|---|
| 1-3个月 | | |
| 3-6个月 | | |
| 6-12个月 | | |
| 1-3年 | | |

必须回答：

- 下一个催化是什么？
- 财报什么时候能体现？
- 订单什么时候能确认？
- 价格什么时候能传导到利润？
- 客户认证什么时候有结果？
- 市场会提前交易还是等业绩兑现？
- 如果兑现晚于预期，估值会不会先回撤？

注意：

- 逻辑正确不等于股价马上涨。
- 长期空间大不等于短期交易赔率高。
- 如果当前市值已经交易远期乐观情景，短期必须看催化密度。

# 九、财务质量

必须检查：

- 收入质量：是否真实增长，是否一次性收入。
- 毛利率：是否可持续。
- 净利率：是否受费用资本化、补贴、投资收益影响。
- 现金流：利润是否变成现金。
- 应收账款：是否大幅增长。
- 存货：是否积压。
- 资本开支：是否重资产。
- 折旧摊销：是否压制利润或隐藏风险。
- 稀释风险：是否频繁融资。
- 负债风险：是否现金流承压。
- 客户集中度：是否依赖单一大客户。
- 订单质量：是否真实可确认收入。
- 利润表和现金流量表是否匹配。

输出：

```text
财务质量结论：高 / 中 / 低。
主要加分项：
主要扣分项：
```

财务质量会影响合理估值倍数。

高财务质量公司可以给更高倍数。  
低财务质量公司即使逻辑好，也要打折。

# 十、交易结构

必须分析交易层面：

- 股价处于低位、突破、加速还是高位分歧？
- 是否经历过充分横盘？
- 成交量是否健康？
- 放量是突破放量、出货放量，还是情绪放量？
- 机构/游资/量化谁可能主导？
- 是否已经成为一致预期？
- 是否有龙虎榜筹码松动？
- 是否有板块共振？
- 是否有指数环境配合？
- 是否适合趋势持有、事件交易、低吸还是只适合观察？

如果用户明确问走势，可联动 `gf-dma-health-index` 或 `horizontal-breakout`。

交易结构必须服务于结论：

```text
基本面逻辑强，但交易结构差，不等于可以买。
交易结构强，但估值已经交易乐观情景，也要防兑现风险。
```

---

# 十一、反证条件

必须写清楚什么情况说明逻辑错了。

常见反证：

- 订单没有兑现。
- 价格没有上涨。
- 产能利用率没有改善。
- 客户认证失败。
- 海外龙头路线变化。
- 财报收入/利润没有跟上。
- 毛利率下滑。
- 现金流恶化。
- 应收和存货异常增长。
- 竞争加剧导致估值逻辑失效。
- 政策/监管导致估值折价扩大。
- 当前市值已经交易乐观情景，但验证仍停留在概念期。
- 海外可比公司估值下修。
- 产业链价格或订单被证伪。
- 公司公告明确澄清业务占比很小或尚未量产。

输出格式：

```text
如果出现以下情况，应下修估值或放弃逻辑：
1. ...
2. ...
3. ...
```

反证条件必须具体，不能只写“业绩不及预期”。

---

# 十二、结论分层

结论必须分成四层：

| 层级 | 结论 |
|---|---|
| 基本面价值 | |
| 估值映射 | |
| 预期差 | |
| 交易赔率 | |

最后给出明确判断：

```text
当前是：
A. 明显低估
B. 合理偏低
C. 合理
D. 合理偏贵
E. 已交易乐观情景
F. 纯主题期权
```

并说明：

- 适合研究还是交易？
- 适合左侧还是右侧？
- 需要等待什么验证？
- 什么情况下可以上修结论？
- 什么情况下必须降级？
- 当前最关键的一个跟踪指标是什么？

# 十三、特殊行业处理

## 13.1 周期股

不能用周期顶部利润线性外推。

必须拆：

- 价格
- 成本
- 产量
- 库存
- 稼动率
- 行业新增产能
- 历史利润分位
- 当前市值对应周期位置

输出：

```text
当前市值对应的是周期底部修复、中枢盈利，还是顶部利润？
```

## 13.2 金融股

优先使用：

- PB-ROE
- PEV
- P/TEV
- NBV
- 投资收益率
- 分红率
- 资本充足率
- 资产质量

必须回答：

```text
当前 PB 隐含多少 ROE？
当前 PEV 隐含多少新业务价值和投资收益率修复？
```

## 13.3 创新药

优先使用：

- 峰值销售
- 适应症空间
- 竞争格局
- 临床阶段
- 成功率折现
- BD 首付款和里程碑
- 海外可比管线估值
- 商业化能力

必须区分：

- 临床前
- I期
- II期
- III期
- NDA/BLA
- 已上市
- 已 BD
- 商业化放量

## 13.4 半导体设备/材料

优先使用：

- 国产替代率
- 客户认证
- 订单
- 产能
- 制程节点
- 毛利率
- 海外可比
- 下游资本开支
- 行业周期

必须区分：

- 已量产供应
- 小批量验证
- 送样
- 研发储备
- 概念沾边

## 13.5 AI软件/SaaS

优先使用：

- ARR
- NRR
- 客户数
- ARPU
- 收入增速
- 毛利率
- 销售费用率
- FCF margin
- 海外可比 PS
- AI 是否增强或削弱商业模式

不能只按“软件股给高 PS”。

## 13.6 军工/卫星/机器人

优先使用：

- 单产品价值量
- 总需求数量
- 公司份额
- 订单确认
- 交付节奏
- 净利率
- 现金流
- 政策和预算周期

必须把市值反推到：

```text
多少台设备 / 多少颗卫星 / 多少套组件 / 多少机器人销量
```

---

# 十四、输出风格要求

- 直接、结论明确。
- 少写泛泛的行业背景，多写“股价隐含什么”。
- 先说结论，再展开推导。
- 表格优先。
- 对不确定数据必须明确标注假设。
- 对当前市场数据必须引用最新来源。
- 对无法确认的信息必须说“不确定”。
- 不要为了显得完整而堆砌无关内容。
- 用户关注交易赔率时，要明确区分基本面判断和交易判断。

---

# 十五、禁止项

- 禁止只说 PE 多少、PS 多少就下结论。
- 禁止只列公司业务，不做市值反推。
- 禁止只说“受益于 AI/国产替代/景气复苏”。
- 禁止不区分老业务和新业务。
- 禁止不写反证条件。
- 禁止不做海外可比。
- 禁止把概念期公司当作业绩兑现公司。
- 禁止把远期 TAM 直接等同于公司收入。
- 禁止在没有当前数据的情况下假装精确。
- 禁止把海外可比公司机械套用到 A 股。
- 禁止忽略财务质量和现金流。
- 禁止忽略当前股价是否已经交易乐观情景。
- 禁止把“逻辑正确”直接等同于“可以买”。
